# Product Brief: LingoRecall AI

## Executive Summary

LingoRecall AI 是一款面向非英语母语者的智能语言学习 Chrome 浏览器扩展。核心价值在于让用户在日常网页浏览中无摩擦地积累词汇，通过 AI 语境分析和艾宾浩斯记忆曲线实现高效学习。

**核心差异化**：一键跳回原文位置并高亮显示 —— 这是现有竞品（沙拉查词、欧路词典、Anki 插件）都未实现的功能。

**战略决策**：基于当前 LingoRecall AI 代码库继续开发（选项 A），借鉴沙拉查词的成熟设计理念。

---

## Core Vision

### Problem Statement

非英语母语者在日常网页浏览中遇到生词时，面临以下核心痛点：

1. **学习摩擦力大**：看到生词 → 开启 AI 助手 → 单独询问 → 问完就忘，流程割裂
2. **脱离语境**：传统词典无法结合当前上下文给出语境化含义，尤其对网络俚语、新词、专业术语
3. **艾宾浩斯遗忘曲线**：没有系统性复习机制，短期记忆无法转化为长期记忆
4. **工具割裂**：手动摘录到 Obsidian/Anki 等工具摩擦力太大，难以坚持

### Problem Impact

- 语言学习者无法有效积累日常浏览中遇到的词汇
- 程序员在 AI 编程时因英语专有词汇理解偏差导致任务分析错误
- 想成为 Twitter 博主但语言不够地道，缺乏系统性积累网络语言的方式
- 学习效果无法量化，缺乏成就感

### Why Existing Solutions Fall Short

#### 竞品分析调研结果（2026-01-06）

| 产品 | 划词翻译 | 语境分析 | 生词本 | **跳回原文** | AI 集成 | 维护状态 |
|------|---------|---------|-------|-------------|---------|----------|
| **沙拉查词** | ✅ | ⚠️ 有限 | ✅ | ❌ **无** | ✅ 新版有 | ⚠️ 停止维护 (2024.4) |
| **欧路词典** | ✅ | ❌ | ✅ | ❌ **无** | ⚠️ 有限 | ✅ 活跃 |
| **Yomitan** | ✅ | ❌ | ✅ (Anki) | ❌ **无** | ❌ | ✅ 活跃 (日语专用) |
| **Migaku** | ✅ | ✅ | ✅ | ❌ **无** | ✅ | ✅ 商业闭源 |

**沙拉查词深度分析 (crimx/ext-saladict)**：
- GitHub Stars: 12.9k | Forks: 821 | License: MIT
- 最后代码提交: 2024年4月6日 (超过1.5年未更新)
- 最后版本发布: 2021年10月17日 v7.20.0 (超过3年!)
- 技术栈: TypeScript + React + RxJS + Webpack (相对老旧)
- 风险: Manifest V3 兼容性存疑，技术债务高

**结论**：没有一款产品实现"跳回原文位置并高亮"功能，这是 LingoRecall AI 的核心差异化机会。

### Proposed Solution

LingoRecall AI 提供以下核心工作流：

1. **无摩擦选词**：在任意网页（Reddit、X、Blog 等）选中词汇，弹出扩展图标
2. **AI 语境分析**：结合上下文，AI 自动生成含义、音标、例句
3. **智能保存**：一键添加到词汇库，支持自定义标签和颜色分类
4. **原文定位**：每个词汇卡保存来源 URL + 选区位置，一键跳回原文并高亮
5. **艾宾浩斯复习**：基于记忆曲线的智能复习提醒
6. **跨平台同步**：配套手机 APP，云同步数据（付费功能）

### Key Differentiators

| 差异化点 | 竞品现状 | LingoRecall AI |
|---------|---------|----------------|
| **原文定位跳回** | ❌ 无人做 | ✅ 一键跳回+高亮 |
| **AI 语境分析** | ⚠️ 简单翻译 | ✅ 结合上下文深度解释 |
| **零成本开源** | 付费为主 | ✅ 本地存储免费 |
| **艾宾浩斯复习** | ❌ 需要 Anki | ✅ 内置复习系统 |
| **Anki 闪卡导出** | ❌ | ✅ 一键生成 |
| **AI 滥用防护** | ❌ | ✅ Token 预算+频率限制 |

## Technical Research & Cost Analysis

### AI API Cost Comparison (2025-2026)

| 模型 | 输入/M tokens | 输出/M tokens | 1000词成本 | 推荐 |
|------|--------------|--------------|-----------|------|
| **Gemini 2.5 Flash-Lite** | $0.10 | $0.40 | **¥0.7** | ⭐ **首选** |
| GPT-4o Mini | $0.15 | $0.60 | ¥1.0 | 备选 |
| Gemini 2.5 Flash | $0.30 | $2.50 | ¥3.9 | 功能更强 |
| GPT-4o | $5.00 | $15.00 | ¥27 | 太贵 |

**成本估算**：
- 每用户每月 1000 词：API 成本约 ¥0.7
- 建议定价：¥5-10/月 或 ¥30-50/年
- 毛利率：>80%

### Storage Solutions

**零成本方案（推荐起步）**：
- `chrome.storage.local` + `unlimitedStorage` 权限：免费，容量无限
- `IndexedDB`：免费，性能更好，支持复杂查询

**付费云同步方案**：
- Firebase/Supabase：~$0.18/GB/月
- 1000用户 × 700KB/用户 ≈ 700MB ≈ $0.13/月

### AI Abuse Prevention Strategies

1. **系统提示词限制**：只回答词汇相关问题
2. **Token 预算**：每用户每天 100 词 / 输出上限 300 tokens
3. **频率限制**：每分钟 10 次请求
4. **上下文绑定**：AI 只能分析选中文本，无开放对话
5. **输入验证**：单词长度 <50 字符，上下文 <500 字符

## Strategic Decision

### 选择方案 A：基于当前 LingoRecall AI 代码库继续开发

**决策理由**：

1. **技术栈最新**：React 19 + TypeScript 5.8 + Vite 6，比沙拉查词先进3代
2. **Manifest V3 兼容**：当前代码已支持，沙拉查词可能需要大幅重构
3. **代码量小**：1,300行 vs 50,000+行，更易维护和扩展
4. **AI 原生集成**：已有 Gemini 服务层，无需重新实现
5. **核心优势保留**："跳回原文位置"功能可直接实现
6. **无技术债务**：干净的代码基础，不需要处理3年未更新的依赖

**借鉴沙拉查词的设计**（不直接 Fork）：
- 划词交互模式设计
- 多词典 API 集成方案
- PDF 划词技术实现
- 生词本数据结构

## MVP Scope

### Core Features（必须实现 - P0）

#### 1. Content Script 网页划词
- 在任意网页中选中文本后显示悬浮操作按钮
- 支持单词和短语选择（限制 50 字符以内）
- 点击按钮触发 AI 分析或保存到词库

#### 2. AI 语境分析
- 结合上下文生成词汇含义解释
- 提供音标、词性、例句
- 针对网络俚语、专业术语的语境化解释

#### 3. 跳回原文高亮（核心差异化）
- 保存词汇时记录：URL + XPath + 选区位置
- 点击"跳回原文"自动打开原页面并滚动定位
- 高亮显示原始选中的单词/短语

#### 4. 本地存储方案
- IndexedDB 主存储 + chrome.storage.local 备用
- 无需登录，零成本起步

#### 5. 艾宾浩斯复习提醒
- 基于记忆曲线计算下次复习时间
- Badge 显示待复习数量
- 简单复习卡片界面

### Out of Scope for MVP（第一版不做）

| 类别 | 功能 | 延迟原因 |
|------|------|----------|
| P1 | 双击查词、悬浮图标 | UI 复杂度，后续添加 |
| P1 | Anki 导出 | 非核心验证 |
| P2 | 多词典聚合 | API 集成工作量大 |
| P2 | PDF 划词、云同步 | 技术复杂度高 |
| 不做 | 手机 APP | MVP 只做 Chrome 扩展 |
| 不做 | 多浏览器支持 | 先验证 Chrome |
| 不做 | 付费功能 | 开源项目，先积累用户 |
| 不做 | 社区功能 | 非核心价值 |
